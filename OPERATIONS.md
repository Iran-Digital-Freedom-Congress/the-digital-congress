## 1. Who Can Join (عضویت)

### Open Participation (No Application Required)
Anyone can join the Slack workspace and observe. The congress is **public by default**.

### Participation Tiers

| Tier | Who | Access | How to Get In |
|---|---|---|---|
| **Observer (ناظر)** | Anyone interested | Read all public channels, watch livestreams | Self-join via invite link |
| **Participant (مشارکت‌کننده)** | Iranians or allies who accept the charter | Write in public channels, join WG discussions, vote in Taraaz | Accept code of conduct on join |
| **Verified Participant (مشارکت‌کننده‌ی تاییدشده)** | Iranians with ZKP identity verification | Vote in formal decisions, participate in sensitive WGs, eligible for facilitation roles | ZKP verification via INID SDK or ZK Passport |
| **Facilitator (تسهیل‌گر)** | Elected/appointed WG leads | Moderate WG channels, set agendas, present at plenaries | Nominated + confirmed by WG members (4-week rotating terms) |
| **Moderator (ناظم)** | Trained conflict resolution volunteers | Enforce code of conduct, mediate disputes, manage channel health | Apply + training + confirmed by Coordination Team |
| **Coordinator (هماهنگ‌کننده)** | Core organizing team | Admin access, strategic decisions, cross-WG coordination | Founding team initially; elected after Month 3 |

### Explicit Welcome
- Iranians of **all ethnicities**: Persian, Kurdish, Azerbaijani, Baloch, Arab, Lur, Turkmen, Armenian, Assyrian, and others
- Iranians of **all political orientations**: republican, federalist, monarchist (constitutional), socialist, liberal, libertarian — as long as they oppose the Islamic Republic
- Iranians **inside and outside Iran**
- **All genders**
- **All generations** — especially young people who led Woman, Life, Freedom
- **Non-Iranian allies** — in observer or supporter capacity
- **Organizational representatives** — parties, NGOs, labor unions, student groups

### Explicit Exclusion (see Red Lines below)

## 2. Red Lines (خطوط قرمز)

### Absolute Red Lines — Immediate Removal

| Red Line | Why | Enforcement |
|---|---|---|
| **Support for the Islamic Republic regime** | This is a freedom congress — regime supporters have the entire state apparatus; this space is not for them | Immediate ban, no appeal |
| **Advocacy for theocratic governance** | Separation of religion and state is a baseline principle | Immediate ban |
| **Doxxing / exposing identities** | Participants inside Iran risk imprisonment or death | Immediate ban + incident report to security team |
| **Sharing operational security details** of activists inside Iran | Endangers lives | Immediate ban |
| **Hate speech targeting ethnic, religious, or gender groups** | Pluralism is foundational | Warning → 24h mute → ban |
| **Calls for ethnic separatism that deny others' rights** | We support self-determination AND territorial discussion, but not supremacism | Case-by-case, mediation first |
| **Harassment, threats, intimidation** | Civic space requires safety | Warning → mute → ban |
| **Spam, commercial promotion, scams** | Keeps the space functional | Immediate removal of content, warning → ban |

### Behavioral Red Lines — Progressive Enforcement

| Behavior | Response |
|---|---|
| **Personal attacks** (attacking the person, not the argument) | Moderator warning → 24h mute |
| **Derailing** (hijacking threads with off-topic content) | Redirect to appropriate channel; repeated = temporary mute |
| **Monopolizing** (one person dominating all conversations) | Facilitator intervention, speaking time limits in video calls |
| **Bad faith argumentation** (strawmanning, whataboutism, trolling) | Moderator warning → mute |
| **Refusing mediation** when in a dispute | Temporary suspension until willing to engage mediator |

### What Is NOT a Red Line (Legitimate Disagreement)
- Debating the future form of government (republic vs. constitutional monarchy vs. federation) — this is **encouraged**
- Disagreeing about the role of foreign intervention
- Criticizing other opposition groups or strategies
- Expressing ethnic/regional identity and demands
- Disagreeing with WG outputs or congress decisions
- Proposing radical ideas that don't violate the above red lines

### The Principle
**"Disagree with ideas freely. Attack people never. Endanger lives never."**

## 3. Moderation Structure (نظارت)

### How Many Moderators?

**Rule of thumb**: 1 moderator per 50-75 active participants, plus 1 per WG channel, plus shift coverage.

| Congress Size | Moderators Needed | Breakdown |
|---|---|---|
| **Launch (50-100 active)** | 6-8 | 2 global + 1 per busy WG channel (rotating) |
| **Growth (100-300 active)** | 10-14 | 3 global + 8 WG + 2 floaters |
| **Scale (300-1000 active)** | 16-22 | 4 global + 8 WG + 4 language-specific + 2 floaters + 2 senior/escalation |
| **Mass (1000+)** | 24+ | Tiered system with junior/senior moderators |

### For Sprint #1 (50 planners + open participants): **8 moderators minimum**

| Role | Count | Responsibility |
|---|---|---|
| **Head Moderator (سرناظم)** | 1 | Final escalation point, sets moderation policy, recruits new mods |
| **Global Moderators** | 2 | Monitor `#general`, `#town-hall`, cross-channel issues |
| **WG Moderators** | 4 | 1 per 2 WG channels (WG1+WG2, WG3+WG4, WG5+WG6, WG7+WG8) |
| **Shift Relief** | 1 | Covers when others need breaks (48h is long) |

### Moderator ≠ Facilitator

| | Moderator (ناظم) | Facilitator (تسهیل‌گر) |
|---|---|---|
| **Focus** | Rule enforcement, safety, channel health | Content, agenda, driving outputs |
| **Acts on** | Code of conduct violations | Discussion flow, time management |
| **Neutrality** | Must be neutral on content | Can have positions, but ensures all voices heard |
| **Decisions** | Warn / mute / ban | Set topics, call votes, summarize |
| **Reports to** | Head Moderator | WG members |

### Moderator Selection & Training
1. **Call for volunteers** — post in `#moderator-recruitment`
2. **Screening**: must accept code of conduct, no affiliation conflicts
3. **Training session** (2 hours before launch):
   - Slack moderation tools
   - De-escalation techniques
   - When to warn vs. mute vs. ban
   - How to handle doxxing / security incidents
   - When to escalate to Head Moderator
4. **Trial period**: first weekend as shadow moderator alongside experienced mod
5. **Review**: after 2 weekends, confirmed or redirected

### Moderation Shifts (for continuous weekends)
- Saturday AM shift: 10:00–13:00
- Saturday PM shift: 13:00–16:00
- Sunday AM shift: 10:00–13:00
- Sunday PM shift: 13:00–16:00
- Weekday async: rotating daily monitor (lighter duty)

**Timezone**: CET/CEST as primary (large diaspora in Europe); but with coverage for US timezones.

### Channel Rules
- **Thread discipline**: Use threads for multi-message responses. Don't flood the main channel.
- **One idea, one thread**: In `#idea-box`, each idea is a single post. Discussion goes in thread.
- **Language**: each WG channel defaults to Farsi with English welcome. Language channels are self-explanatory.
- **Off-topic**: `#off-topic` exists for a reason. Use it. Moderators will redirect non-topical content.
- **No voice messages in text channels**: Keep text searchable and accessible.

|---|---|---|---|

| **Telegram** | Share in major Iranian diaspora groups + channels | Outreach team | Weekly + on-demand |

| **YouTube** | Town hall recordings, 5-min weekly summary clips | Livestream team | Weekly |

### First Saturday of Every Month: Open House
- Dedicated onboarding session (1 hour before regular programming)
- "What is DIFC? How to participate" walkthrough
- New members meet WG facilitators
- Assign a "buddy" (existing member) to each newcomer

## 6. Organizing Thoughts & Ideas (سازمان‌دهی ایده‌ها)

### The Idea Pipeline

```
                  ┌─────────────────┐
                  │   #idea-box     │  ← Anyone posts, any time
                  │   (raw ideas)   │     Simple format: Title + 3 sentences
                  └────────┬────────┘
                           │ Moderator tags: theme + relevant WG
                           ▼
                  ┌─────────────────┐
                  │ #idea-discussion│  ← Threaded discussion
                  │  (refinement)   │     Community reacts, asks questions
                  └────────┬────────┘
                           │ If idea gains traction (5+ positive reactions)
                           ▼
                  ┌─────────────────┐
                  │  WG Channel     │  ← Relevant WG adopts the idea
                  │  (development)  │     WG develops it during weekend session
                  └────────┬────────┘
                           │ WG produces a draft proposal
                           ▼
                  ┌─────────────────┐
                  │  #proposals     │  ← Formal proposal (template)
                  │  (structured)   │     Title, Problem, Solution, Resources,
                  └────────┬────────┘     Timeline, Success Metrics
                           │ Taraaz deliberation or vote
                           ▼
                  ┌─────────────────┐
                  │  Taraaz / Vote  │  ← Community deliberation + decision
                  │  (decision)     │
                  └────────┬────────┘
                           │ If approved
                           ▼
                  ┌─────────────────┐
                  │  Implementation │  ← Team forms, resources allocated,
                  │  (action)       │     tracked in #documentation
                  └─────────────────┘
```

### #idea-box Post Format (keep it dead simple)
```
💡 [IDEA TITLE]
What: [one sentence — what you're proposing]
Why: [one sentence — what problem it solves]
How: [one sentence — rough approach]
Tag: #wg1 #wg3 (which working groups might care)
```

That's it. No long documents. No proposal templates at this stage. Lower the barrier to minimum.

### #proposals Template (for developed ideas)
```
📋 PROPOSAL: [Title]

PROBLEM: What's broken / missing?
SOLUTION: What do we do about it?
OWNER: Who takes responsibility?
RESOURCES: What do we need? (people, money, tools)
TIMELINE: When does it start and what are milestones?
SUCCESS: How do we know it worked?
RISKS: What could go wrong?
RELATED: Links to prior discussion in #idea-box / WG channels

STATUS: [Draft / Under Deliberation / Approved / In Progress / Complete]
```

### Collaborative Idea Generation Methods

| Method | When | How | Tool |
|---|---|---|---|
| **Taraaz / Polis** | Weekly town halls | Post a statement, everyone agrees/disagrees/passes. Algorithm finds consensus clusters. | taraaz.jomhoor.org |
| **Brainstorm threads** | Anytime in WG channels | Facilitator posts a prompt. 10-minute timed thread. No criticism during brainstorm phase. | Slack threads |
| **Reverse brainstorm** | When stuck | "What would make this problem WORSE?" Then invert. | Slack or video call |
| **Idea tournaments** | Monthly | 8 ideas enter, community votes in bracket format, 1 winner gets implementation support | Slack polls + Taraaz |
| **Expert AMAs** | Bi-weekly | Invite a subject matter expert (lawyer, economist, engineer, activist). Open Q&A. | Slack + video call |
| **Cross-WG jams** | Monthly | Two WGs merge for a session to find intersection ideas (e.g., WG3 Finance + WG1 Diplomacy) | Video call |
| **Silent writing** | During WG sessions | 10 min: everyone writes individually. Then share. Prevents groupthink. | Google Docs / HackMD |
| **Dot voting** | After brainstorms | Everyone gets 3 dots (emoji reactions) to place on their favorite ideas | Slack reactions |

### Documentation & Knowledge Base

Everything the congress produces is documented and searchable:

| What | Where | Format | Updated |
|---|---|---|---|
| Meeting notes | `#documentation` + Git wiki | Markdown | After every session |
| WG outputs | Git repository | Markdown / PDF | Weekly |
| Proposals | `#proposals` channel + Git | Template format | As submitted |
| Taraaz results | Linked from `#taraaz-polls` | Polis reports | After each deliberation |
| Decisions log | Pinned in `#announcements` | Simple table: Date / Decision / WG / Vote count | After each decision |
| Idea tracker | Shared spreadsheet or Notion | Status pipeline (raw → discussed → adopted → proposed → approved → implemented) | Weekly |

### The Key Principle: **Ideas Flow Up, Power Flows Down**
- Anyone can submit an idea (bottom-up)
- WGs develop ideas collaboratively (peer-level)
- Community decides via Taraaz/voting (democratic)
- Implementation teams self-organize (subsidiarity)
- **No one person or group can veto an idea that the community supports**

## 7. Quick Reference: Launch Week Moderation Setup

### Day 1 Before Launch
- [ ] Recruit 8 moderators (post call on X/Telegram, direct outreach)
- [ ] 2-hour moderator training session (Zoom/Jitsi)
- [ ] Set up `#moderator-room` (private) with playbook pinned
- [ ] Configure Slack moderation tools (auto-moderation for slurs, ban list)

### Moderator Playbook (pinned in #moderator-room)

```
ESCALATION LADDER:

1. GENTLE REDIRECT
   "Hey, this might fit better in #[channel]. Mind moving it?"
   
2. CONTENT WARNING
   "Reminder: personal attacks violate our code of conduct.
    Please focus on the argument, not the person."

3. FORMAL WARNING (DM to user)
   "This is a formal warning. [Specific behavior]. 
    Next violation → 24h mute."

4. TEMPORARY MUTE (24h)
   Apply Slack restriction. Notify in #moderator-room.
   User can still read, not write.

5. EXTENDED MUTE (1 week)
   For repeat offenders. Requires Head Moderator approval.

6. BAN
   Permanent removal. Requires Head Moderator approval + documentation.
   
⚡ EMERGENCY (doxxing, security breach, threats):
   Skip all steps. Immediate ban. Alert #security. 
   Notify Head Moderator within 15 minutes.
```
